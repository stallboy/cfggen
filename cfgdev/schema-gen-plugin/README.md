# cfggen-architect

游戏架构师与数据驱动配置生成 Claude Code 插件。根据策划文档或自然语言描述，生成符合 cfggen 规范的模块化 schema 定义文件。

## 安装

## 使用方法

直接向 Claude 描述您的配置需求即可：

```
创建一个物品表，包含 id、名称、类型、价格、描述
```

```
设计一个任务系统，包含多种完成条件（击杀怪物、收集物品）和奖励
```

```
根据目录xx下的策划文档，生成配表
```

## 战斗系统设计

当需求涉及战斗/技能系统时，技能会自动参考 `skill-system-design.md`，该文档定义了：

- **GameplayTag** - 层级化标签系统
- **Ability/Effect/Status** - 三层架构
- **FloatValue** - 动态求值接口
- **Trigger** - 事件触发器模式
- **事件管线** - Pre/Post 两阶段提交

### 各系统简评

* **星际争霸2 ABE (38分)**：纯数据驱动的鼻祖，机制拼装极其自由。**痛点**：引用关系如同蜘蛛网，逻辑与表现绑定较深，调试和上手是噩梦级别。
* **虚幻引擎 GAS (40分)**：目前的 3A 工业标杆，解耦做到极致，自带极其完善的网络预测。**痛点**：重度依赖 C++ 框架，学习曲线极其陡峭，小型项目极易过度设计。
* **早期 CFG (35分)**：务实的古典方案。通过树状结构解决了传统 Excel 配置的死板问题，引入了 `Context` 和 `Blackboard` 来处理复杂的参数流转 。**痛点**：存在较多硬编码耦合（如显式枚举 `SendDamageCfg` 等具体业务事件），且逻辑节点中混杂了表现层配置（如命中特效 `hitSfxs`）。
* **现代 CFG GAS (42分)**：极其成功的现代化重构。完美吸收了 GAS 的精髓，全面转向 **Tag 驱动**和 **EventBus 载荷修饰**，实现了逻辑与表现的彻底剥离。其底层的 `SafeList` 和 `TagContainer` 优化甚至比一些商业引擎更轻量优雅。**痛点**：配置逻辑高度抽象，策划的认知门槛逼近程序级别。

总而言之，现代 CFG GAS 文档中的现代架构远超 PDF 中的传统方案，是一套非常优秀的工业级设计准则。
